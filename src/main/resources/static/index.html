Overview

Live feed

The backend exposes a STOMP endpoint at /ws and a topic broker under /topic. Use your preferred client to subscribe and receive live updates in real time.

History

REST endpoints under /api/records/history/aggregated return time-bucketed data. Adjust your query parameters to filter by sensor type and desired time windows.

MQTT

Devices publish messages to the configured MQTT broker. The backend consumes those payloads, normalizes them, and persists both raw and aggregated values for downstream consumers.
